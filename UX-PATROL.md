# UX Patrol — 20분 순회 가이드

> 이 문서는 자동 UX 순회 cron이 참조하는 작업 가이드입니다.
> 수정 시 CLAUDE.md의 디자인 시스템 및 코드 규칙을 반드시 준수하세요.

## 목표
- 200~300명이 동시에 퀴즈 풀고, 투표하고, 리액션 보낼 때 **렉 없이** 부드러운지 확인
- 학생/강사/스태프 간 **피드백 루프가 끊기지 않는지** 확인
- 기능적 동작 오류, 크래시, 데드 스테이트 탐지
- 발견된 이슈는 즉시 수정 + 커밋 + firebase deploy

## 핵심 원칙
- **사용자 관점**: 코드가 아니라 화면을 본다. "학생이 이걸 보면 뭘 해야 하는지 알까?"
- **실전 시나리오**: 300명 강의실, 불안정한 Wi-Fi, 폰 한 손 조작, 강사는 프로젝터 앞
- **문제 감지 → 즉시 수정**: 다음 순회까지 미루지 않음

## 순회 순서

### 0. 사전 준비
- `npm run build` 성공 확인 (실패 시 즉시 수정)
- dev 서버 실행 확인 (`http://localhost:5173`)
- 콘솔 에러/경고 확인

### 1. 라이브 퀴즈 시나리오 (가장 중요!)

> 300명이 동시에 퀴즈를 풀 때의 시나리오. 렉, 멈춤, 화면 깨짐이 없어야 함.

**1-1. 퀴즈 출제 → 학생 투표 → 정답 공개**
- 강사가 퀴즈 질문 활성화 → 학생 화면에 즉시 질문 표시되는지
- 선택지(A/B/C/D) 클릭 → "전송 중..." 표시 → VoteConfirm 전환 부드러운지
- 선택 후 다른 선택지 fade-out 애니메이션 자연스러운지
- VoteConfirm → "결과를 기다리는 중..." 전환 3초 타이밍 적절한지
- 실시간 집계 바 "실시간 집계 · 확정 전" 헤더 표시되는지

**1-2. 타이머 카운트다운**
- 타이머 작동 시 학생 화면에서 카운트다운 보이는지
- 5초 이하일 때 펄스 효과 있는지
- 만료 시: 이미 투표한 학생 → 오버레이 안 뜸 (VoteConfirm 유지)
- 만료 시: 미투표 학생 → "응답하지 못했습니다" 오버레이

**1-3. 정답 공개 + 결과 화면**
- 강사 "정답 공개" → 학생에게 QuizResult 표시되는지
- 정답 맞힘: 체크 아이콘 + 파티클 + "정답!" + 점수 카운트업
- 정답 틀림: X 아이콘 + 흔들림 + "오답" + 정답 표시
- **맞혔을 때도 정답이 뭔지 표시되는지** (학습 효과)
- 점수 변동 자연스러운지 (CountUp 애니메이션)
- 연속 정답 시 불꽃 배지 표시

**1-4. 리더보드**
- 강사 "리더보드" 전환 → 학생 화면에 리더보드 표시
- 내 순위 카드(MyRankCard) 상단 고정
- 순위 변동 화살표 8초간 표시
- 1등 왕관 애니메이션
- 10위 밖 학생: 하단 고정 순위 표시
- 순위 변동 시 행(row) 재정렬 애니메이션 부드러운지

**1-5. 스피드퀴즈 연속 모드**
- 자동 진행: 질문 → 투표 → 정답 공개 → 다음 질문
- 전환 사이사이 깜빡임/멈춤 없는지
- 콤보 카운터 표시 (3연속 이상)
- 진행 표시 "X / Y" 배너

### 2. 실시간 상호작용 시나리오

**2-1. 리액션 폭탄**
- 200명이 동시에 리액션 → 버블 애니메이션 부드러운지
- 프레젠터 화면에서 떠오르는 효과 렌더링 괜찮은지
- 쿨다운(3초) 동안 버튼 비활성 표시 있는지

**2-2. 채팅 폭주**
- 200명이 채팅 → 스크롤 부드러운지
- 새 메시지 도착 시 자동 스크롤
- 쿨다운(2초) 동안 버튼 비활성

**2-3. 손들기 동시 다수**
- 50명이 동시에 손들기 → 강사 패널에서 목록 렌더링 괜찮은지
- 강사가 "전체 해제" → 학생들에게 "선생님이 확인했어요" 토스트

**2-4. 긴급 질문 다수**
- 20명이 긴급 질문 전송 → 강사 패널 목록 쌓임
- 강사가 질문 클릭 → reviewing 배너 학생 표시
- 강사가 "확인" → acknowledged 배너 → 3초 후 삭제

### 3. 강사↔학생 피드백 루프 체크

> "강사가 행동했는데 학생이 모르는" 상황이 없어야 함

| 강사 액션 | 학생에게 보여야 하는 것 | 체크 |
|-----------|----------------------|------|
| 질문 활성화 | 투표 화면 전환 | 즉시 |
| 질문 비활성화 | 대기 화면 복귀 | 즉시 |
| 정답 공개 | QuizResult 표시 | 즉시 |
| 리더보드 전환 | 리더보드 화면 | 즉시 |
| 타이머 시작 | 카운트다운 표시 | 즉시 |
| 손들기 해제 | "선생님이 확인했어요" 토스트 | 3초 내 |
| 긴급 질문 확인 | "강사가 질문을 확인했습니다" 배너 | 3초 내 |
| DM 해결 완료 | "OO님이 도움을 완료했습니다" 토스트 | 3초 내 |
| 세션 종료 | SessionEndedPage 표시 | 즉시 |
| 발표 모드 전환 | (학생 무관, 프레젠터만) | - |

### 4. 학생 하단바 전체 기능 점검

| 버튼 | 동작 | 체크 포인트 |
|------|------|------------|
| 손들기 | 토글 on/off | 버튼 스타일 변화, 펄스 링, "내리기" 텍스트 |
| 긴급질문 | 모달 열림 | 익명 토글, 300자 제한, 전송 토스트 |
| 질문(Q&A) | 패널 슬라이드 | 질문 작성, 추천, 답변 달기 |
| 채팅 | 패널 슬라이드 | 메시지 입력, 자동 스크롤, 안 읽은 알림 |
| 도움 | DM 모달 | 채팅방 탭, 새 요청 탭, 대기중/진행중/해결됨 |

### 5. 모바일 UX 점검

> 학생의 80%는 폰으로 접속. 한 손 조작에 최적화되어야 함.

- 하단바 버튼 48px+ 터치 타겟
- 키보드 열릴 때 입력 필드 가려지지 않는지
- 스크롤 시 헤더/하단바 고정 유지
- safe-area-inset (노치/홈바) 적용
- 투표 버튼이 thumb zone(하단 1/3)에 있는지

### 6. 프레젠터 화면 점검

> 프로젝터/대형 화면에 띄우는 화면. 뒷자리에서도 보여야 함.

- QR 코드 크기 충분한지
- 실시간 투표 결과 바 애니메이션 부드러운지
- 리더보드 글자 크기 충분한지
- 워드클라우드/OX배틀 등 시각화 렌더링
- 게임(룰렛, 뽑기 등) 실행 시 렉 없는지

### 7. 에러/엣지 케이스 체크

- 빌드 에러 0건
- 콘솔 에러 0건, key warning 0건
- 네트워크 끊김 → "연결 끊김" 배너 (3초 디바운스)
- 재연결 → "다시 연결됨" 배너 + online 상태 복구
- 시크릿 모드 접속 → 크래시 없이 동작
- 페이지 새로고침 → 투표 상태 복원
- 세션 종료 후 학생 → SessionEndedPage (조작 불가)

### 8. 성능 지표 체크

- 번들 크기 급증 없는지 (VotePage <120KB, AdminPage <185KB)
- Suspense fallback이 0.5초 이상 보이지 않는지
- AnimatePresence 전환 시 깜빡임 없는지
- 리스트 렌더링 (참여자, 채팅, Q&A) 스크롤 부드러운지

## 판단 기준

### 최적화가 기능을 망가뜨리지 않았는지 체크
> "빠르게 만들었지만 동작이 안 되면 의미 없다"

- 투표 선택 후 "전송 중..." → VoteConfirm 전환이 실제로 일어나는지 (submitting 상태 도입 후)
- 타이머 만료 오버레이가 이미 투표한 학생에게 안 뜨는지 (useMyVote 도입 후)
- 손들기 해제 시 자기가 내린 경우 토스트 안 뜨는지 (selfToggledRef 도입 후)
- DM 해결 토스트가 초기 로드 시 잘못 뜨지 않는지 (prevStatusRef 초기값 {} 확인)
- 긴급 질문 acknowledged 후 3초 뒤 실제로 삭제되는지 (deleteTimerRef 도입 후)
- 연결 배너 디바운스가 너무 길어서 진짜 끊김을 못 알려주진 않는지 (3초 적정)
- Wi-Fi 재연결 시 online: true가 실제로 복구되는지 (syncPresence 도입 후)
- localStorage try/catch가 정상 브라우저에서도 문제없이 읽기/쓰기 되는지

### 즉시 수정 (P0)
- 크래시/화면 터짐
- 버튼 클릭 후 반응 없음 (dead click)
- 강사 액션 → 학생 무반응 (피드백 루프 끊김)
- 투표/퀴즈 기능 오작동
- 콘솔 에러
- 빌드 실패
- 렉/프레임 드롭이 체감되는 수준

### 다음 순회에서 수정 (P1)
- 애니메이션 어색함 (동작은 하지만 부자연스러운)
- 텍스트/라벨 오타
- 다크모드 스타일 누락
- 접근성(aria) 누락

### 수정하지 않는 것
- 취향 차이 수준의 디자인 변경
- 스키마/DB 구조 변경이 필요한 것
- 새로운 기능 추가
- 이미 300명 부하테스트 통과한 성능 이슈

## 수정 시 규칙
1. CLAUDE.md 디자인 시스템 준수 (slate 모노크로매틱, lucide 아이콘, 한국어)
2. 최소 변경 — 관련 코드만 수정, 리팩토링 금지
3. `npm run build` 통과 확인
4. git commit (한국어 메시지, 변경 요약)
5. `git push origin main`
6. `npx firebase deploy --only hosting`
7. 커밋 내용 요약 로그 출력

## 발견 없을 때
"no findings — 빌드 OK, 퀴즈/투표/리액션/피드백루프 정상" 한 줄 로그

## 매 순회 시 랜덤으로 1개 이상 깊이 점검

아래 중 매번 다른 항목을 골라서 코드 레벨까지 확인:
- [ ] ChoiceVoter: 선택 → 전송 → 확인 전체 흐름 코드 리뷰
- [ ] QuizVoter: 베팅 → 자신감 → 투표 → 결과 흐름
- [ ] OXVoter: O/X 선택 → 결과
- [ ] SpeedQuiz: 자동 진행 흐름 (타이머 → 공개 → 다음)
- [ ] Leaderboard: 순위 재정렬 + 애니메이션
- [ ] DMBubble: 학생 도움 요청 → 스태프 응답 → 해결
- [ ] HandRaise: 학생 손들기 → 강사 확인 → 학생 토스트
- [ ] UrgentQuestion: 학생 질문 → 강사 확인 → 학생 배너
- [ ] ChatPanel: 메시지 전송 → 수신 → 스크롤
- [ ] TimerCountdown: 시작 → 카운트다운 → 만료 처리
- [ ] ReviewingBanner: reviewing → acknowledged → 완료 배너
- [ ] ConnectionBanner: 끊김 디바운스 → 재연결 표시
- [ ] SessionEndedPage: 세션 종료 → 최종 화면
- [ ] PresentationView: 프레젠터 화면 전체
- [ ] ReactionOverlay: 리액션 버블 애니메이션
